# v0.4 及之前黑盒测试报告

> 测试日期：2026-05-07  
> 测试方式：从使用者/消费者角度进行黑盒验证，覆盖 v0.4 及之前全部用户可见能力。  
> CCW 会话：`.workflow/.ccw-coordinate/CCW-20260507-111104/`

## 结论

**整体判定：未通过。**

v0.4 及之前的核心规则与数据包有较多局部通过证据，包括角色、骰子、战斗、法术、物品、地图和剧本数据加载。但从真实使用者路径看，项目当前不能完成“安装依赖 -> 构建 -> 启动后端 -> 打开前端 -> 创建会话 -> 进行角色/战斗/法术/物品/地图交互”的完整闭环。

主要阻塞是后端 WebSocket 包编译失败、前端生产构建失败、根类型检查失败、前端黑盒验收存在 Save 反馈回归，以及 `pnpm-lock.yaml` 与根 `package.json` 不一致导致 frozen install 失败。

## 阻塞问题

### P0：后端 WebSocket 编译失败，服务无法启动

命令：

```bash
cd apps/server
go test ./...
```

结果：

```text
internal/api/websocket/handler_llm.go:243:7: duplicate case "start_combat"
internal/api/websocket/handler_llm.go:142:7: previous case
internal/api/websocket/handler_llm.go:290:7: duplicate case "end_combat"
internal/api/websocket/handler_llm.go:153:7: previous case
```

证据位置：

- `apps/server/internal/api/websocket/handler_llm.go:142`
- `apps/server/internal/api/websocket/handler_llm.go:153`
- `apps/server/internal/api/websocket/handler_llm.go:243`
- `apps/server/internal/api/websocket/handler_llm.go:290`

用户影响：后端不能构建、测试或启动，WebSocket 主路径、战斗开始/结束事件、前端会话联动都无法对外交付。

### P0：前端生产构建失败

命令：

```bash
pnpm --filter web build
```

关键错误：

- `apps/web/src/components/panels/QuickActions.tsx:1`：`useRef` 未使用。
- `apps/web/src/components/panels/SpellbookPanel/index.tsx:168`：`character.knownSpells` 可能为 `undefined`。
- `apps/web/src/game/entities/BaseEntity.ts:15`：`combatant` 未初始化。
- `apps/web/src/game/entities/Enemy.ts:30`：`Combatant` 上不存在 `cr`。
- `apps/web/src/game/scenes/CombatScene/CombatEventHandler.ts`、`CombatStoreSyncer.ts`：访问 `BaseScene.addUnsubscribe` protected 成员。
- `apps/web/src/game/ui/NameLabel.ts:16`：`setName` 返回类型与 Phaser `Text.setName` 不兼容。
- `apps/web/src/hooks/useGameMessages.ts:76`：`phase` 的 `string` 类型不能赋给 `GamePhase`。

用户影响：前端不能发布生产包，v0.4 的法术/物品/地图/战斗 UI 无法作为可发布版本交付。

### P0：冷安装失败

命令：

```bash
pnpm install --frozen-lockfile
```

结果：

```text
ERR_PNPM_OUTDATED_LOCKFILE
specifiers in the lockfile don't match specifiers in package.json:
* 1 dependencies were removed: npm-run-all@^4.1.5
```

用户影响：CI 默认 frozen lockfile 或新用户按锁文件安装时会直接失败。

### P1：根类型检查失败

命令：

```bash
pnpm type-check
```

结果：`@dnd-game/shared` 执行 `tsc --noEmit` 后只打印 TypeScript 帮助并退出 1。该包 `packages/shared/package.json:9` 定义了 `type-check`，但缺少可让 `tsc` 检查的有效项目配置或输入。

用户影响：根级质量门禁不可用，即使 `web` 单包 type-check 通过，整个工作区仍不能通过。

### P1：前端黑盒 Save 反馈验收失败

命令：

```bash
pnpm --filter web exec vitest run src/tests/blackbox-acceptance.test.tsx
```

结果：21 个用例中 2 个失败。

失败用例：

- `AC2: Save button visible feedback > should change button label from "Save" to "Saved!" after click`
- `AC2: Save button visible feedback > should revert button label back to "Save" after 2 seconds`

测试期望位于 `apps/web/src/tests/blackbox-acceptance.test.tsx:146` 和 `apps/web/src/tests/blackbox-acceptance.test.tsx:160`：点击 Save 后按钮 title/文本变为 `Saved!`，2 秒后恢复。

当前实现位于 `apps/web/src/components/panels/QuickActions.tsx:247` 到 `apps/web/src/components/panels/QuickActions.tsx:327`：点击后 `saveState` 变为 `saving`，按钮仍使用 `action.label` 作为 title 和文本，即 `Save`；反馈显示在按钮下方 `Saving adventure...` / `Save complete`，且成功状态依赖后端通知。

用户影响：保存动作缺少测试约定的即时可见反馈，玩家不能从按钮本身确认保存完成。

## 风险与观察

- v0.4 后端法术、物品、地图、剧本基础能力有局部测试通过，但 WebSocket 编译失败使这些能力无法通过完整服务路径交付。
- 前端 dev server 可打开页面壳，但后端不可启动时 `/api/sessions` 和 WebSocket 交互会失败，不能代表完整玩家体验可用。
- 根级 e2e/integration 入口存在大量 skipped，用例通过数不能证明完整 v0.4 用户路径已覆盖。
- 文档中 v0.3/v0.4 仍有“计划中”状态描述，而代码中已有实现与测试，建议后续报告区分“文档验收标准”“代码测试证据”“运行证据”。

## 建议修复顺序

1. 修复 `apps/server/internal/api/websocket/handler_llm.go` 中重复的 `start_combat` / `end_combat` case，保留一处完整状态快照与事件发送逻辑，然后重跑 `cd apps/server && go test ./...`。
2. 同步 `pnpm-lock.yaml` 与根 `package.json`，确保 `pnpm install --frozen-lockfile` 通过。
3. 修复前端生产构建的 TypeScript 错误，重跑 `pnpm --filter web build`。
4. 修复 `packages/shared` 的 type-check 配置，使 `pnpm type-check` 在根目录通过。
5. 对齐 Save 反馈实现与黑盒验收：要么按钮即时切换为 `Saved!` 并按测试恢复，要么更新验收测试到当前 UX 设计，但需保证用户可见反馈明确。
6. 在后端可启动后补做真实闭环 smoke：启动 server + web，验证创建会话、创建角色、开始战斗、施法、使用/装备物品、地图/剧本加载。
