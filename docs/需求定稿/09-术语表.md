# 术语表

> [返回索引](./index.md)

---

| 术语 | 定义 |
| --- | --- |
| **D&D 5e** | 龙与地下城第5版规则，本系统的游戏规则基础 |
| **DM** | Dungeon Master（地下城主），游戏主持人，本系统中由 LLM 担任 |
| **MCP** | Model Context Protocol，Client 与 Server 之间的通信协议 |
| **LLM** | Large Language Model，大语言模型 |
| **游戏后端（Backend）** | Client 模块与 Server 模块的整合，提供完整的游戏服务 |
| **FVTT** | Foundry Virtual Tabletop，本系统采用其地图格式 |
| **React** | Facebook 开源的前端 UI 框架，用于构建 Web 界面 |
| **Phaser** | HTML5 游戏框架，用于 Canvas 游戏内容渲染（战斗场景、地图等） |
| **WebSocket** | 全双工通信协议，用于 Web 前端与游戏后端之间的实时通信 |
| **角色卡** | 玩家角色的完整数据记录，包含种族、职业、属性、技能等 |
| **生物卡** | NPC 和敌人的统一数据结构，包含属性、能力、行为模式等 |
| **冒险地图** | 世界大地图，显示大致范围、玩家当前位置、各地点信息 |
| **场景地图** | FVTT 格式详细地图，支持移动、交互、遮挡、墙壁、光照 |
| **战棋模式** | 回合制战斗，角色在地图上移动，参考博德之门 3 |
| **上下文** | LLM 会话中保留的全部信息，分为剧本数据、运行时状态、历史会话、规则参考、用户设置五大类 |
| **剧本数据** | 从 PDF 剧本解析得到的静态只读数据 |
| **运行时状态** | 随游戏进行不断变化的动态数据 |
| **渐进式披露** | 按三层结构向 LLM 披露工具：索引 → 激活 → 执行 |
| **检定** | D&D 中通过掷骰子来确定某项行动是否成功的机制 |
| **DC** | Difficulty Class，难度等级 |
| **优势/劣势** | 两次掷骰取高/低 |
| **HD** | Hit Dice，生命骰，用于短休息恢复 HP |
| **AC** | Armor Class，护甲等级 |
| **CR** | Challenge Rating，挑战等级 |
| **遭遇** | 剧本中设计的战斗、社交、解谜等事件 |
| **事件触发器** | 满足特定条件时自动触发的游戏事件 |
| **实时同步** | 数据变更时立即同步，非定时心跳 |
| **实时推送** | Server/Client 状态变化主动推送到 Web 前端 |
