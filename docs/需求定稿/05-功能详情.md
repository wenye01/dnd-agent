## 5.1 上下文系统

上下文是 LLM 进行决策和叙述时所能访问的全部信息。

### 5.1.1 上下文内容分类

| 类别 | 特性 | 描述 |
| --- | --- | --- |
| **剧本数据** | 静态 | 从 PDF 剧本解析得到的固定内容 |
| **游戏运行时状态** | 动态 | 随游戏进行不断变化的状态 |
| **历史会话** | 累积 | 对话记录、行动历史、事件记录 |
| **规则参考** | 静态 | D&D 5e 规则摘要、剧本特定规则 |
| **用户设置** | 持久 | 用户偏好、显示设置、快捷键等 |

### 5.1.2 上下文切换机制

- **目的**：管理 token 消耗，优化 LLM 性能
- **触发条件**：
  - **自动触发**：进入战斗时自动切换到战斗上下文
  - **LLM 决定**：其他情况由 LLM 决定是否切换
- **上下文类型**：
  - **探索上下文**：默认状态，包含角色卡、位置、章节目标等
  - **战斗上下文**：进入战斗时自动切换，包含战斗属性、敌人数据、战斗规则等
  - **对话上下文**：与重要NPC深度对话时，包含NPC完整数据、对话历史等

### 5.1.3 历史会话压缩

- **目的**：控制上下文窗口大小
- **策略**：预留压缩接口，具体实现待后续阶段设计

## 5.2 战斗系统（战棋模式）

- **参考**：博德之门 3
- **核心特点**：
  - 战斗在场景地图上进行
  - 支持角色移动、距离计算
  - 掩护判定、机会攻击
  - 墙壁、光照影响战斗
- **规则执行**：
  - Server 严格执行 D&D 5e 规则
  - **不允许**执行规则外的操作
- **调用方式**：
  1. 细粒度：LLM 依次调用检定、掷骰子等
  2. 高层抽象：LLM 调用攻击，Server 自动完成全部流程

---

## 5.3 实时数据同步

- **机制**：有修改即同步（非传统心跳）
- **同步内容**：全部游戏状态
- **恢复**：调用失败时可以进行重试，等待重新连接，退出后使用持久化数据恢复

## 5.4 地图系统

| 地图类型 | 用途 | 特点 |
| --- | --- | --- |
| **冒险地图（WorldMap）** | 世界大地图 | 显示大致范围、玩家当前位置、各地点信息 |
| **场景地图（SceneMap）** | 具体场景 | FVTT 格式，支持移动、交互、遮挡、墙壁、光照 |

**场景地图特性**：
- 标签系统：不同地点可进入不同场景
- FVTT 数据：墙壁、光照、网格
- 交互元素：参考博德之门 3 的 RPG 交互

**FVTT 使用范围**：

| 功能 | 是否使用 | 说明 |
| --- | --- | --- |
| **地图数据（墙壁、光照、网格）** | ✅ 使用 | 核心地图数据 |
| **音频系统** | ❌ 不使用 | 当前版本不使用 |
| **宏系统** | ❌ 不使用 | 当前版本不使用 |
| **Token 系统** | 部分使用 | 参考数据结构，自行实现角色显示 |
| **光源数据** | ✅ 使用 | 保留数值，对战斗探索无影响，供后续版本使用 |

## 5.5 Web 前端界面

Web 前端是用户与游戏系统交互的核心界面，采用 **React + Phaser** 双层架构。

### 5.5.1 技术架构

| 层级 | 技术栈 | 职责 |
| --- | --- | --- |
| **UI 层** | React | 用户输入框、角色面板、对话历史、Server 监控、管理操作等界面元素 |
| **Canvas 层** | Phaser | 冒险地图、场景地图、战斗场景、角色移动动画、攻击动画、光照效果等游戏内容 |

**通信方式**：
- Web 前端与游戏后端之间使用 **WebSocket** 进行实时双向通信
- 支持状态推送、事件订阅、用户输入传递

### 5.5.2 地图展示（Phaser Canvas 层）

| 地图类型 | 展示内容 | 交互方式 |
| --- | --- | --- |
| **冒险地图** | 世界概览、地点标记、玩家当前位置、区域信息 | 点击地点查看详情/进入 |
| **场景地图** | FVTT 格式详细地图、墙壁、光照、交互元素、角色位置 | 点击移动、与元素交互 |
| **战斗地图** | 战棋网格、角色站位、移动范围、攻击范围、掩护显示 | 点击选择目标/确认移动 |

### 5.5.3 用户输入（React UI 层）

| 输入类型 | 描述 | 示例 |
| --- | --- | --- |
| **自然语言** | 文本输入框，用于描述角色行动 | "我向守卫询问关于失踪商人的事" |
| **地图点击** | 在场景地图上点击进行移动或交互 | 点击门 → 开门 |
| **快捷操作** | 常用操作的快捷按钮 | 攻击、施法、使用物品、检定、结束回合 |
| **战斗操作** | 战斗回合内的行动选择 | 选择目标、确认攻击、选择法术 |

### 5.5.4 战斗可视化（Phaser Canvas 层）

| 可视化元素 | 描述 |
| --- | --- |
| **战棋网格** | 显示角色站位、移动范围、攻击范围 |
| **回合指示** | 当前行动角色高亮、回合顺序显示 |
| **移动动画** | 角色移动的平滑动画效果 |
| **攻击动画** | 攻击、法术的视觉效果（使用简单动画即可） |
| **伤害数字** | 伤害/治疗数值的弹出显示 |
| **状态效果** | 增益/减益状态的图标和持续时间显示 |
| **骰子动画** | 检定时的骰子滚动动画（可选） |

### 5.5.5 Server 监控与管理（React UI 层）

- **监控**：实时展示 Server 内部执行过程（检定 → 掷骰子 → 执行）
- **管理**：支持管理操作（强制结束战斗、修改数据等）
- **架构**：低耦合，工具注册机制

### 5.5.6 信息展示（React UI 层）

| 展示内容 | 位置/形式 |
| --- | --- |
| **角色面板** | 侧边栏，显示属性、HP、状态效果、背包等 |
| **对话历史** | 主界面区域，DM 叙述和对话记录 |
| **系统消息** | 通知区域，检定结果、系统提示等 |
| **剧本信息** | 可展开面板，当前任务、已知地点、NPC 信息等 |
| **存档管理** | 存档列表、新建/加载/删除存档、存档命名（可选） |

## 5.6 剧本系统

- **剧本格式**：PDF 需预先解析成格式化数据
- **剧本内容**：
  - 地点信息
  - NPC 数据
  - 事件触发条件
  - 战斗遭遇
  - 剧情节点
- **当前限制**：
  - 暂不支持用户自定义剧本
  - **剧本解析工具暂不实现**，当前版本使用手动预处理的数据
