## 3.1 功能清单

### 3.1.1 核心功能 (P0)

| 功能名称 | 描述 | 涉及角色 |
| --- | --- | --- |
| **用户输入处理** | 接收用户自然语言，传递给 LLM | 用户, Client, LLM |
| **LLM 工具调用解析** | 解析 LLM 返回的工具调用指令，执行相应操作 | Client |
| **会话管理** | 创建/管理游戏会话，完整持久化（对话+游戏状态） | Client, Server |
| **创建游戏** | 创建新的游戏会话，返回 sessionId | Client, Server |
| **游戏状态查询** | 查询当前游戏状态，返回缺失信息提示 | Server |
| **角色创建** | 按 D&D 5e 规则创建角色（支持多角色队伍） | Client, Server, LLM |
| **选择剧本** | 从内置剧本中选择，加载必要信息到上下文 | Client, Server |
| **DM 叙述** | LLM 作为 DM 进行故事叙述 | LLM, 用户 |
| **AI 角色控制** | LLM 控制 NPC 和敌人的行为 | LLM, Server |
| **战斗系统** | D&D 5e 战棋式战斗（参考博德之门 3） | Server |
| **检定/掷骰子** | 属性检定、技能检定、攻击检定等 | Server |
| **地图移动** | 场景地图上的角色移动，距离计算 | Server |
| **地图交互** | 与场景地图元素交互（开门、拾取、触发陷阱等） | Server |
| **Web 地图展示** | 在 Web 前端展示冒险地图和场景地图 | Web, Server |
| **Web 用户输入** | Web 端自然语言输入框、地图点击交互 | Web, Client |
| **Web 战斗可视化** | 战棋网格、角色移动、攻击动画、伤害显示 | Web, Server |

### 3.1.2 重要功能 (P1)

| 功能名称 | 描述 | 涉及角色 |
| --- | --- | --- |
| **多 LLM 支持** | 支持 OpenAI 和 Anthropic API 格式 | Client |
| **上下文切换** | 进入战斗自动切换，其他由 LLM 决定 | Client |
| **实时数据同步** | Client-Server 游戏状态实时同步 | Client, Server |
| **冒险地图** | 世界大地图，显示玩家位置和地点信息 | Server |
| **场景地图** | FVTT 格式详细地图，墙壁、光照、交互元素 | Server |
| **背包系统** | 物品管理、装备穿戴 | Server |
| **法术系统** | 法术学习、准备、施放 | Server |
| **状态效果** | 增益/减益效果管理 | Server |

### 3.1.3 增强功能 (P2)

| 功能名称 | 描述 | 涉及角色 |
| --- | --- | --- |
| **Web 监控** | 实时展示 Server 内部执行过程 | Server, Web |
| **Web 管理** | 通过 Web 界面进行管理操作 | Web |
| **Web 角色面板** | 角色属性、背包、法术的可视化展示和操作 | Web, Server |
| **Web 对话历史** | DM 叙述、对话记录的展示 | Web, Client |
| **Web 地图交互** | 点击地图移动、与地图元素交互 | Web, Server |
| **Web 战斗动画** | 攻击、法术、状态效果的可视化动画 | Web, Server |
| **Web 快捷操作** | 常用操作的快捷按钮（攻击、施法、使用物品、检定、结束回合） | Web, Client |

## 3.2 D&D 5e 规则功能

| **属性** | 六大属性（力量、敏捷、体质、智力、感知、魅力）及修正值 |
| **技能** | 技能熟练、专长、工具熟练 |
| **背景** | 角色背景特性 |
| **等级** | 升级、属性提升、新特性获得 |
| **专长** | 专长选择与效果 |

| 子系统 | 功能点 |
| --- | --- |
| **先攻** | 先攻检定、先攻顺序 |
| **行动** | 行动、附赠行动、反应、自由动作 |
| **移动** | 移动速度、困难地形、跳跃、攀爬、游泳 |
| **攻击** | 攻击检定、伤害骰、暴击、失手 |
| **护甲等级** | AC 计算、护甲类型、盾牌 |
| **伤害与治疗** | HP、临时 HP、治疗、伤害类型、伤害抗性/免疫 |
| **战棋规则** | 距离、触及、掩护、机会攻击 |

## 3.3 核心业务流程

### 3.3.1 创建游戏流程

```
1. 用户输入开始游戏的自然语言描述
2. Client 接收文本，调用 LLM
3. LLM 理解用户需求，调用"创建会话"工具
4. Client 解析工具调用，请求 Server 创建会话
5. Server 创建会话，返回 sessionId 及其他信息
6. Client 使用 sessionId 创建本地会话
7. 实时同步 Client-Server 游戏状态
8. 创建完成
```

### 3.3.2 创建角色流程

```
1. 游戏创建成功后，LLM 引导用户创建角色
2. 用户回复角色创建信息（种族、职业、属性等）
3. Client 传递给 LLM
4. LLM 调用 Server 工具创建角色
5. Server 按 D&D 5e 规则验证并创建角色
6. 返回成功/错误信息（错误时提示缺少/错误内容）
7. 可重复创建多个角色（队伍模式）
8. 实时同步游戏状态
```

### 3.3.3 选择剧本流程

```
1. Web 前端呈现可选剧本列表
2. 用户在 Web 前端选择剧本
3. Web 前端将选择发送给 Client 模块
4. Client 接收选择，发送给 LLM
5. LLM 调用 Client 的加载剧本功能
6. Client 加载剧本，必要信息载入上下文
```

### 3.3.4 游戏过程流程

```
循环:
  1. LLM 作为 DM 叙述背景/过程，或控制 NPC 行动
  2. 用户做出行动（或 LLM 控制的敌人行动）
  3. Client 接收后发给 LLM
  4. LLM 根据输入选择:
     - 继续叙述
     - 回答用户问题
     - 调用 Server 工具（检定、技能、交互等）
  5. Server 严格执行 D&D 5e 规则，不允许规则外操作
  6. 如果进入战斗:
     a. 自动切换到战斗上下文
     b. 进入战棋模式（参考博德之门 3）
     c. 按先攻顺序循环
     d. 战斗结束后恢复上下文
  7. 实时同步所有状态变化
```

### 3.3.5 地图切换流程

```
1. AI DM 决定进入场景
2. 根据场景标签加载对应的场景地图
3. 进入场景地图，角色放置到入口位置
4. 在场景地图中移动、交互
5. 离开场景返回冒险地图
```
