## 1. 子系统概述

### 1.1 子系统定位

Web 前端子系统是用户与 D&D AI DM 系统交互的核心界面，负责：

- 用户交互接收与反馈
- 游戏世界可视化（地图、角色、战斗）
- 实时状态展示
- 与游戏后端的通信

### 1.2 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| React | 18.x | UI 组件框架 |
| TypeScript | 5.x | 类型安全 |
| Phaser | 3.70+ | 游戏引擎 |
| Zustand | 4.x | 状态管理 |
| Tailwind CSS | 3.x | 样式方案 |
| Vite | 5.x | 构建工具 |

## 2. 模块间通信

### 2.1 通信关系图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           模块间通信关系                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│                         ┌─────────────────┐                            │
│                         │     UI 模块     │                            │
│                         └────────┬────────┘                            │
│                                  │                                      │
│              ┌───────────────────┼───────────────────┐                 │
│              │                   │                   │                 │
│              ▼                   ▼                   ▼                 │
│   ┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐    │
│   │   状态管理模块   │◀─│   游戏引擎模块   │  │     工具模块     │    │
│   │                  │  │                  │  │                  │    │
│   │   (读写状态)     │─▶│   (读取状态)     │  │   (提供工具)     │    │
│   └────────┬─────────┘  └──────────────────┘  └──────────────────┘    │
│            │                                                            │
│            │ 状态更新                                                   │
│            ▼                                                            │
│   ┌──────────────────┐                                                 │
│   │     通信模块     │◀═════════════════════════════════════════════╗  │
│   │                  │                                               ║  │
│   │   (同步状态)     │═════════════════════════════════════════════╝  │
│   └──────────────────┘          WebSocket 消息                        │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 通信方式

| 源模块 | 目标模块 | 通信方式 | 说明 |
|--------|----------|----------|------|
| UI 模块 | 状态管理模块 | Zustand API | 读写应用状态 |
| UI 模块 | 通信模块 | 函数调用 | 发送用户操作 |
| 游戏引擎模块 | 状态管理模块 | Zustand API | 读取游戏状态 |
| 游戏引擎模块 | UI 模块 | 事件回调 | 触发 UI 更新 |
| 通信模块 | 状态管理模块 | Zustand API | 更新服务端推送的状态 |
| 所有模块 | 工具模块 | 函数导入 | 使用工具函数和类型 |

## 4. 模块依赖关系

### 4.1 依赖层次

```
层次 4: UI 模块
    │
    ├──▶ 层次 3: 游戏引擎模块
    │
    ├──▶ 层次 2: 状态管理模块
    │
    └──▶ 层次 1: 通信模块
              │
              └──▶ 层次 0: 工具模块
```

### 4.2 依赖规则

1. **上层依赖下层**：高层模块可以依赖低层模块
2. **禁止反向依赖**：低层模块不能直接依赖高层模块
3. **跨层通过接口**：跨层依赖通过明确的接口进行
4. **工具模块独立**：工具模块不依赖任何其他模块

## 5. 设计原则

### 5.1 模块边界原则

- **单一职责**：每个模块只负责一个明确的功能领域
- **高内聚**：模块内部组件紧密相关
- **低耦合**：模块间通过明确接口通信，减少隐式依赖
- **可替换性**：模块可以独立测试和替换

### 5.2 状态管理原则

- **单一数据源**：后端是状态的唯一真实来源
- **单向数据流**：UI → Action → Store → UI
- **增量更新**：接收差异更新，减少传输量
- **乐观更新**：关键操作需要后端确认

### 5.3 通信原则

- **消息驱动**：模块间通过消息/事件通信
- **异步处理**：网络操作异步处理，不阻塞 UI
- **错误隔离**：单个模块错误不影响其他模块
- **重试机制**：网络失败自动重试
