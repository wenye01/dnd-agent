### 1.1 版本目标

在 v0.5 基础上，打磨用户体验，补充边缘功能，完善错误处理，确保系统稳定可靠。

### 1.2 核心功能

| 功能领域 | 实现内容 |
|----------|----------|
| **规则完善** | 补充边缘规则、死亡豁免完整实现 |
| **错误处理** | 完善错误恢复、重试机制、状态回滚 |
| **管理接口** | 强制结束战斗、数据修改、调试工具 |
| **视觉优化** | 光照效果、动画优化、UI 打磨 |
| **性能优化** | 渲染优化、内存管理、资源加载 |

### 1.3 发布标准

- [ ] D&D 5e 核心规则完整覆盖
- [ ] 稳定的游戏体验（无崩溃）
- [ ] 完善的错误处理和恢复
- [ ] 良好的用户界面体验

death.FailOnDamage(characterId, isCritical)
// 暴击: 失败 2 次
// 普通: 失败 1 次

// 状态检查
status := death.GetStatus(characterId)
// {successes: 1, failures: 2, state: "dying"}

**验收标准**：
1. 死亡豁免正确执行
2. 暴击规则正确
3. 受伤失败正确

#### B403 - 错误恢复系统

| 属性 | 描述 |
|------|------|
| **任务ID** | B403 |
| **任务名称** | 错误恢复系统完善 |
| **所属模块** | client, persistence |
| **优先级** | P0 |
| **依赖** | v0.5 所有任务 |

**功能描述**：
- 实现操作回滚机制
- 实现状态快照恢复
- 实现 LLM 调用失败恢复
- 实现乐观锁和冲突解决

**实现目的**：
确保系统可以从错误中恢复，不丢失用户进度。

**预期效果**：
```go
// 操作回滚
err := stateManager.AtomicUpdate(func(tx *Transaction) error {
    // 一系列操作
    if err != nil {
        tx.Rollback() // 自动回滚
        return err
    }
    return nil
})

// 状态快照
snapshot := stateManager.CreateSnapshot()
// ... 操作失败时 ...
stateManager.RestoreSnapshot(snapshot)

// LLM 调用恢复
retry.WithRetry(llmCall, 3, backoff.Exponential)
```

**验收标准**：
1. 操作失败可以回滚
2. 快照可以正确恢复
3. LLM 调用重试成功率高

**可用资料**：
- [技术路线图 - LLM 调用失败处理](../tech-roadmap.md#15-数据存储方案)

## 5. 验收测试场景

### 场景 1：死亡和复活

1. 角色 HP 降为 0
2. 开始死亡豁免
3. 成功 3 次稳定 / 失败 3 次死亡
4. 死亡后通过法术或服务复活
5. 验证复活惩罚正确应用

### 场景 2：错误恢复

1. 进行一系列操作
2. 模拟 LLM 调用失败
3. 系统自动重试
4. 验证状态一致性

### 场景 3：性能测试

1. 长时间游戏会话（2小时+）
2. 大量战斗和地图切换
3. 验证内存稳定
4. 验证帧率稳定
