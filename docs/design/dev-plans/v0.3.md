# v0.3 开发计划 - 战斗系统

> **版本**: v0.3
> **状态**: 计划中
> **前置版本**: v0.2 (角色创建 + 基础UI)
> **目标版本**: v0.4 (法术+物品+地图)

---

## 1. 版本概述

### 1.1 版本目标

在 v0.2 基础上，实现完整的角色系统和战斗系统，使游戏具备完整的战斗体验。

### 1.2 核心功能

| 功能领域 | 实现内容 |
|----------|----------|
| **角色系统** | 完整种族/职业/背景数据、CharacterCreator 服务 |
| **战斗系统** | 完整战斗流程（先攻、回合、攻击、伤害、结束） |
| **状态效果** | 15 种标准状态效果（P1） |
| **休息恢复** | 短休息/长休息/死亡豁免基础 |

### 1.3 发布标准

- [ ] 完整的角色创建流程可正常运行（种族/职业/背景数据完整）
- [ ] 完整的战斗流程可正常运行
- [ ] 死亡豁免基础实现
- [ ] 战斗动画和场景正常工作

// 移除状态效果
effects.Remove(target, Condition)

// 检定修正
modifier := effects.GetModifier(target, AbilityCheck)
// blinded: 视觉相关检定劣势
// prone: 近战攻击劣势
```

1. 15种状态效果正确定义

// 成功 3 次：稳定
// 失败 3 次：死亡

// 稳定
death.Stabilize(characterId)
```

**验收标准**：
1. 短休息正确恢复
2. 长休息正确恢复
3. 死亡豁免正确执行

## 5. 验收测试场景

### 场景 1：完整角色创建

1. 用户通过自然语言创建角色
2. 选择完整种族（含种族特性）
3. 选择完整职业（含职业特性）
4. 角色属性和 HP 正确计算

### 场景 2：完整战斗流程

1. 用户通过自然语言触发战斗
2. 系统初始化战斗地图和参战单位
3. 按先攻顺序轮流行动
4. 玩家选择攻击目标
5. 系统计算攻击和伤害
6. 敌人行动
7. 战斗结束，显示结果

### 场景 3：死亡豁免

1. 玩家 HP 降为 0
2. 系统触发死亡豁免检定
3. 成功 3 次：角色稳定
4. 失败 3 次：角色死亡
