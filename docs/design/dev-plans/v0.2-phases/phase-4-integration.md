## 测试场景

### 场景 1：完整角色创建流程

**步骤**：
1. 启动后端服务（端口 8080）
2. 启动前端开发服务器（端口 5173）
3. 浏览器访问 http://localhost:5173
4. 点击"开始游戏"
5. 进入游戏界面
6. 检查 WebSocket 连接状态（应显示"已连接"）
7. 在聊天框输入："我想创建一个精灵法师"
8. 观察流式 DM 叙述
9. 观察角色面板更新

**验收点**：
- [ ] WebSocket 连接正常
- [ ] 用户消息可发送
- [ ] DM 流式回复正常显示
- [ ] LLM 正确调用 `create_character` 工具
- [ ] 角色面板显示新创建的角色
- [ ] 角色属性计算正确

---

### 场景 2：会话恢复

**步骤**：
1. 创建一个角色
2. 与 DM 进行几轮对话
3. 刷新页面
4. 验证状态恢复

**验收点**：
- [ ] 会话状态正确恢复
- [ ] 消息历史保留
- [ ] 角色信息仍然显示
- [ ] 可以继续对话

---

### 场景 3：属性检定

**步骤**：
1. 在聊天中请求进行属性检定（如"我尝试打开这扇门"）
2. 观察 DM 调用 `roll_ability_check` 工具
3. 验证骰子结果显示

**验收点**：
- [ ] LLM 正确识别需要检定
- [ ] 骰子工具正确执行
- [ ] 检定结果正确返回
- [ ] DM 根据结果叙述

---

### 场景 4：游戏状态查询

**步骤**：
1. 请求查看当前游戏状态
2. 验证 `get_game_state` 工具调用

**验收点**：
- [ ] 游戏状态正确返回
- [ ] 包含角色信息
- [ ] 包含场景信息
