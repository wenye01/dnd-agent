## 开发原则

### 最小可用架构

1. **核心骨架**：先搭建可运行的最小框架
2. **垂直切片**：实现一个完整的端到端功能
3. **可验证**：每个版本都可以运行和验证

### 小步快跑

```
v0.1: 基础设施（骨架+通信+LLM+持久化）
  │
  ▼
v0.2: 角色创建 + 基础 UI（可与 DM 对话，创建角色）
  │
  ▼
v0.3: 战斗系统（完整战斗流程）
  │
  ▼
v0.4: 法术 + 物品 + 地图（完整游戏体验）
  │
  ▼
v0.5: 剧本 + 世界地图 + 存档
  │
  ▼
v1.0: 死亡复活，错误恢复，UI 打磨
```

### 前后端分离

每个版本分为前后端两个独立计划：
- 后端计划指导 Go 代码实现
- 前端计划指导 React/Phaser 代码实现
- 两者可并行开发

## 开发指南

### 开发流程

1. **阅读设计文档**: 先阅读相关设计文档
2. **理解任务要求**: 查看任务总览和验收标准
3. **实现功能**: 按照实现指南编写代码
4. **编写测试**: 编写单元测试
5. **验收测试**: 通过验收标准

### 前后端并行开发

```
后端开发者                          前端开发者
    │                                  │
    │  v0.2 后端计划                   │  v0.2 前端计划
    │  ├── 角色系统基础                │  ├── 基础 UI 组件
    │  ├── 工具注册                    │  ├── 聊天界面
    │  └── 会话管理                    │  ├── 角色面板
    │                                  │  └── 主界面布局
    └──────────── 集成测试 ────────────┘
```
