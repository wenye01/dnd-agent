# 开发计划 - v0.2 角色创建 + 基础 UI

> **版本**: v0.2
> **状态**: 计划中
> **前置版本**: v0.1 (基础设施)
> **目标版本**: v0.3 (战斗系统)

---

## 1. 版本概述

### 1.1 版本目标

在 v0.1 基础上，实现角色系统、工具注册、会话管理，以及完整的前端 UI，让用户可以通过自然语言创建角色并看到角色信息。

### 1.2 核心功能

| 功能领域 | 实现内容 |
|----------|----------|
| **角色系统** | 角色创建基础（3种族/职业原型） |
| **工具注册** | 5个核心 MCP 工具（roll_dice, create_character 等） |
| **会话管理** | LLM 会话、消息历史、工具调用流程 |
| **前端 UI** | 聊天界面、角色面板、主界面布局 |
| **游戏引擎** | Phaser 集成（P1，v0.3 前必须完成） |

### 1.3 发布标准

- [ ] 用户可以通过自然语言创建角色
- [ ] LLM 可以进行基础 DM 叙述（工具调用正常）
- [ ] 前端可以显示消息和基础角色信息
- [ ] 游戏状态可以通过会话管理持久化

## 2. 功能任务列表

### 2.1 后端任务

#### B004 - 角色系统基础

| 属性 | 描述 |
|------|------|
| **任务ID** | B004 |
| **任务名称** | 角色系统基础实现 |
| **所属模块** | server/character |
| **优先级** | P0 |

**功能描述**：
- 定义角色数据模型（Character, NPC, Enemy）
- 实现角色创建（种族、职业、背景、属性）
- 实现属性计算（属性修正、熟练加值、技能加值）
- 实现基础属性（HP、AC、速度、先攻）

**实现目的**：
提供角色创建和属性管理的基础能力，支持用户通过自然语言创建 D&D 角色。

**预期效果**：
```go
// 角色创建
char := character.Create(character.CreateParams{
    Name:   "艾拉",
    Race:   "elf",
    Class:  "wizard",
    Background: "sage",
    AbilityScores: map[Ability]int{STR: 8, DEX: 14, CON: 12, INT: 16, WIS: 12, CHA: 10},
})

// 属性计算
char.AC           // 12 (10 + DEX mod)
char.Proficiency  // +2
char.Skills[Perception] // +3 (WIS mod + proficiency)
```

**验收标准**：
1. 支持常用种族（人类、精灵、矮人）和职业（战士、法师、盗贼）作为原型验证；完整种族/职业数据在 v0.3 B212 中实现
2. 属性计算符合 D&D 5e 规则（修正符、熟练加值、技能加值）
3. 角色可正确序列化为 JSON
4. 单元测试覆盖核心计算逻辑

**可用资料**：
- [角色系统模块设计](../modules/backend/character.md)
- [角色系统组件设计](../modules/backend/components/character-components.md)
- [D&D 5e SRD - Character Creation](https://www.dndbeyond.com/sources/basic-rules/step-by-step-characters)

#### B006 - Client 会话管理

| 属性 | 描述 |

| **任务名称** | Client 会话管理实现 |
| **所属模块** | client/session, client/context |
| **优先级** | P0 |

**功能描述**：
- 实现会话生命周期管理（创建、恢复、销毁）
- 实现消息历史管理
- 实现基础上下文构建（System Prompt + 消息历史）
- 实现工具调用处理流程

**实现目的**：
管理 LLM 会话状态，构建发送给 LLM 的上下文，处理 LLM 的工具调用请求。

**预期效果**：
```go
// 创建会话
session := client.NewSession(scenarioID)

// 发送用户消息
response := session.SendMessage(ctx, "我想创建一个精灵法师")

// 处理工具调用
for _, call := range response.ToolCalls {
    result := toolRegistry.Execute(call.Name, call.Arguments)
    session.AddToolResult(call.ID, result)
}
```

**验收标准**：
1. 会话可以正确初始化和恢复
2. 消息历史可以正确追加
3. 上下文构建符合 LLM 输入格式
4. 工具调用结果可以正确返回给 LLM

**可用资料**：
- [Client 模块设计](../modules/backend/client.md)

#### B007 - Server 工具注册

| 属性 | 描述 |
|------|------|
| **任务ID** | B007 |
| **任务名称** | Server 工具注册和执行 |
| **所属模块** | client/tools |
| **优先级** | P0 |

**功能描述**：
- 实现工具注册表
- 定义核心工具（roll_dice, create_character, get_character 等）
- 实现工具执行和结果返回
- 实现工具 Schema 定义（JSON Schema）

**实现目的**：
注册 Server 模块的功能为 LLM 可调用的工具，实现 Client 到 Server 的调用桥梁。

**预期效果**：
```go
// 注册工具
registry.Register(ToolDefinition{
    Name: "roll_dice",
    Description: "Roll dice using standard notation",
    InputSchema: {...},
    Handler: diceTool.Roll,
})

// 执行工具
result := registry.Execute("roll_dice", map[string]interface{}{
    "formula": "4d6k3",
})
```

**验收标准**：
1. 工具 Schema 符合 OpenAI Function Calling 格式
2. 工具执行结果可以正确序列化
3. 错误可以正确返回给 LLM
4. 至少实现 5 个核心工具：`roll_dice`、`create_character`、`get_character`、`get_game_state`、`roll_ability_check`

**可用资料**：
- [后端模块设计 - MCP 工具定义](../modules/backend/index.md#5-mcp-工具定义)
- [接口设计文档 - MCP 工具列表](../interfaces.md)

2. 流式响应可以逐字输出
3. 工具调用格式正确转换
4. 可通过配置文件切换 LLM Provider
