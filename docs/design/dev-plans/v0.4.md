# v0.4 开发计划 - 法术+物品+地图

> **版本**: v0.4
> **状态**: 计划中
> **前置版本**: v0.3 (战斗系统)
> **目标版本**: v0.5 (剧本+世界地图+存档)

---

## 1. 版本概述

### 1.1 版本目标

在 v0.3 基础上，实现法术系统、物品系统、地图系统，使游戏具备完整的探索和施法体验。

### 1.2 核心功能

| 功能领域 | 实现内容 |
|----------|----------|
| **法术系统** | 法术位、施法、专注管理 |
| **物品系统** | 背包、装备、物品使用 |
| **地图系统** | 场景地图加载、交互点、触发器 |
| **剧本系统** | 剧本加载、章节管理（基础） |
| **上下文管理** | 探索/战斗/对话上下文自动切换（P1） |

### 1.3 发布标准

- [ ] 法术施放和专注管理正常工作
- [ ] 背包物品管理正常工作
- [ ] 场景地图可正确加载和渲染
- [ ] 剧本可正确加载

effect := spell.GetEffect(spellId)
// 魔法飞弹: 3d4+3 力场伤害
// 治疗术: 1d8+施法属性 治疗量

#### B211 - 上下文管理完善

| 属性 | 描述 |
|------|------|
| **任务ID** | B211 |
| **任务名称** | 上下文管理完善 |
| **所属模块** | client/context |
| **优先级** | P1 |
| **依赖** | v0.2 B006, v0.3 B201 |

**功能描述**：
- 实现探索上下文（默认状态，包含角色卡、位置、章节目标）
- 实现战斗上下文（进入战斗时自动切换，包含战斗属性、敌人数据、战斗规则）
- 实现对话上下文（深度对话 NPC 时，包含 NPC 完整数据、对话历史）
- 实现上下文自动切换逻辑

**实现目的**：
根据游戏状态动态调整发送给 LLM 的上下文内容，在控制 Token 消耗的同时保证 LLM 有足够的决策信息。

**预期效果**：
```go
// 上下文构建器
type ContextType string
const (
    ContextExploration ContextType = "exploration"
    ContextCombat      ContextType = "combat"
    ContextDialog      ContextType = "dialog"
)

// 自动切换（进入战斗时）
ctx := contextBuilder.Build(gameState)
// gameState.IsInCombat() → 自动选择战斗上下文
// gameState.IsInDialog(npcId) → 自动选择对话上下文

// System prompt 根据上下文类型不同
systemPrompt := ctx.BuildSystemPrompt()
```

**验收标准**：
1. 进入战斗时自动切换到战斗上下文
2. 战斗结束后自动恢复探索上下文
3. 各上下文包含正确的信息集合
4. System prompt 内容正确反映当前上下文类型

**可用资料**：
- [需求定稿 - 上下文系统](../../需求定稿/05-功能详情.md#51-上下文系统)
- [Client 模块设计](../modules/backend/client.md)

## 5. 验收测试场景

### 场景 1：法术施放

1. 玩家选择施放法术
2. 系统检查法术位
3. 玩家选择目标
4. 系统计算法术效果
5. 法术消耗法术位
6. 专注法术添加到专注列表

### 场景 2：物品使用

1. 玩家打开背包
2. 选择物品使用
3. 物品效果应用（恢复 HP 等）
4. 物品从背包移除

### 场景 3：场景地图交互

1. 玩家在地图上点击移动
2. 角色走向目标位置
3. 点击交互点（门/宝箱/NPC）
4. 触发相应事件
